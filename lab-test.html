Network-lab-test — Quick Lab Test

Simple, copyable commands for beginners: Nmap scans, tshark/Wireshark captures, and a quick Burp reminder. Use these in your VM lab (Kali or similar).

How to use

  1. Open your lab VM (Kali). Ensure network is the target subnet (e.g. 192.168.131.0/24).
  2. Choose a tool tab on the right, click Copy and paste into your VM terminal.
  3. Save results into scans/ folder and review in Wireshark or with the provided templates.

Notes

Only scan networks you own or have permission to test. The examples below are sanitized for training.

Common workflows

  • Discover hosts: fast SYN scan across subnet.
  • Service enumeration: version detection on discovered ports.
  • Capture traffic: use tshark or Wireshark to validate live flows.

Commands & Quick Copy
Install (Debian/Kali)
sudo apt update && sudo apt install -y nmap
Full SYN scan (requires sudo)
sudo nmap -sS -p- -T4 -oA scans/full_192_168_131_0_24 192.168.131.0/24
Service/version scan (example host)
sudo nmap -sV -sC -p 22,80,139,443 -oN scans/192.168.131.10_services.txt 192.168.131.10
Tshark capture (60s)
sudo tshark -i any -a duration:60 -f "net 192.168.131.0/24" -w scans/capture_subnet.pcap
Open in Wireshark (GUI)
wireshark scans/capture_subnet.pcap
Common display filter
ip.addr==192.168.131.0/24 && (tcp.port==22 || tcp.port==80 || tcp.port==443 || tcp.port==139)
Start Burp Suite (if installed)
burpsuite
